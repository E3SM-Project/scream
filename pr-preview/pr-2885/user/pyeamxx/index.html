    PyEAMxx

    PyEAMxx is a C-Python interoperability layer that allows to interact with EAMxx from Python. As of today (Jun 28th 2024), we support only the interaction with a single physics parametrization from Python. The user can create the parametrization, initialize it, and run it, as well as modify the values of its parameters and input fields.

    While we are planning on packaging pyeamxx into a conda-installable package, as of today the only way to get pyeamxx support is to manually build EAMxx with the CMake option EAMXX_ENABLE_PYBIND=ON. The module will be build in <build_dir>/src/python.

    The following is an example of how to run P3 from python

    import sys
sys.path.append(/path/to/eamxx/build/dir/src/python')

import pyeamxx
import numpy as np

# Without these, and manual init/finalize, you may get some
# obscure MPI errors at exit.
import mpi4py
mpi4py.rc.initialize = False  # do not initialize MPI automatically
mpi4py.rc.finalize = False    # do not finalize MPI automatically

#########################################
def main ():
#########################################

    # Timestepping params
    nsteps = 10
    dt     = 600                # seconds
    t0_str = 2021-10-12-45000   # YYYY-MM-DD-XXXXX

    ic_file = Path('/path/to/eamxx/ic/file.nc')

    # Create the grid. Note: ncols MUST match the dim 'ncol' in the IC file
    ncols = 218
    nlevs = 72
    pyeamxx.create_grids_manager(ncols,nlevs,str(ic_file))

    # Create input parameters, and then create the process
    params_dict = {}
    params_dict['max_total_ni'] = 740.0e+3
    params_dict['do_prescribed_ccn'] = False

    p3 = pyeamxx.AtmProc(params_dict,'p3')

    # Initialize input fields
    missing = p3.read_ic(str(ic_file))
    if len(missing)>0:
        print (f"WARNING! The following input fields were not found in the IC file, and must be manually initialized: {missing}")

    # Get a handle of the C++ parameter list
    p3params = p3.get_params()

    # Run p3 for a set of values of 'p3_rho_rime_min'
    param_vals = [50.0 75.0 100.0]
    surf_temp = {}
    for mu_r in param_vals:
        p3.initialize(t0_str)
        for n in range(0,nsteps):
            p3.run(dt)

        # Store surface temperature value at column 0
        T_mid = p3.get_field('T_mid')
        T_mid_array = T_mid.get()
        surf_temp[mu_r] = T_mid_array[0,nlevs-1]


    # Run p3 several times, adding a small perturbation to the initial value of T_mid
    ens_size = 10
    T_mid = p3.get_field('T_mid')
    T_mid_array = T_mid.get()
    for i in range(0,ens_size):
        p3.read_ic(str(ic_file))
        perturb = np.random.normal(0,1e-3,nlevs)
        T_mid_array[0,:] *= perturb
        p3.initialize(t0_str)

        for n in range(0,nsteps):
            p3.run(dt)

        # Store surface temperature value at column 0
        T_mid = p3.get_field('T_mid')
        T_mid_array = T_mid.get()
        surf_temp[mu_r] = T_mid_array[0,nlevs-1]

####################################
if  __name__  == "__main__":
    # This level of indirection ensures all pybind structs are destroyed
    # before we finalize eamxx (and hence kokkos)
    MPI.Init()
    pyeamxx.init()
    main ()
    pyeamxx.finalize()
    MPI.Finalize()

    Additionally, the user can request a single output stream from the atm process, by calling p3.setup_output(X) after calling p3.initialize(t0_str). Here, X is either a string for the name of the yaml file containing the I/O parameters, or the dictionary that would result by parsing such a file. The latter is useful if running several ensemble members, so that we can load the yaml file once, and then update the name of the output file for each ensemble member. The syntax for the yaml file is the same as for regular EAMxx runs, and it is covered in the model output page of this documentation.
